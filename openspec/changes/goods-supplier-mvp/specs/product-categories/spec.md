## ADDED Requirements

### Requirement: Создание категории
Система SHALL позволять создавать новую категорию товаров с обязательным полем `name` и опциональным полем `description`.

#### Scenario: Успешное создание категории
- **WHEN** пользователь отправляет POST запрос на `/api/categories` с `name`="Ингредиенты"
- **THEN** система создаёт категорию и возвращает статус 201 с данными категории

#### Scenario: Создание категории без названия
- **WHEN** пользователь отправляет POST запрос на `/api/categories` без поля `name`
- **THEN** система возвращает статус 400 с описанием ошибки

### Requirement: Получение списка категорий
Система SHALL возвращать список всех категорий. Каждая категория MUST содержать `id`, `name`, `description` и количество связанных поставщиков.

#### Scenario: Получение всех категорий
- **WHEN** пользователь отправляет GET запрос на `/api/categories`
- **THEN** система возвращает массив всех категорий с количеством поставщиков в каждой

### Requirement: Обновление категории
Система SHALL позволять обновлять поля категории по её идентификатору.

#### Scenario: Успешное обновление
- **WHEN** пользователь отправляет PUT запрос на `/api/categories/1` с новым `name`="Упаковка и тара"
- **THEN** система обновляет категорию и возвращает обновлённые данные

#### Scenario: Категория не найдена
- **WHEN** пользователь отправляет PUT запрос на `/api/categories/999`
- **THEN** система возвращает статус 404

### Requirement: Удаление категории
Система SHALL позволять удалять категорию. Категория MAY быть удалена только если с ней не связано ни одного поставщика.

#### Scenario: Удаление пустой категории
- **WHEN** пользователь отправляет DELETE запрос на `/api/categories/1` и категория не имеет связанных поставщиков
- **THEN** система удаляет категорию и возвращает статус 200

#### Scenario: Удаление категории с поставщиками
- **WHEN** пользователь отправляет DELETE запрос на `/api/categories/1` и к категории привязаны поставщики
- **THEN** система возвращает статус 400 с сообщением о невозможности удаления

### Requirement: Предустановленный набор категорий
Система SHALL предоставлять начальный набор категорий при первой инициализации: Ингредиенты, Готовая продукция, Упаковка, Оборудование, Напитки, Специи и приправы.

#### Scenario: Начальные данные
- **WHEN** система инициализируется впервые
- **THEN** в базе данных доступны предустановленные категории
