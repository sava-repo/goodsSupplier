## ADDED Requirements

### Requirement: Получение данных для сравнения
Система SHALL позволять получать данные нескольких поставщиков одновременно для их сравнения по идентификаторам.

#### Scenario: Сравнение нескольких поставщиков
- **WHEN** пользователь отправляет GET запрос на `/api/suppliers/compare?ids=1,2,3`
- **THEN** система возвращает полные данные поставщиков с id=1, id=2 и id=3 в массиве

#### Scenario: Один из поставщиков не найден
- **WHEN** пользователь отправляет GET запрос на `/api/suppliers/compare?ids=1,999`
- **THEN** система возвращает данные только для найденного поставщика id=1, игнорируя id=999

#### Scenario: Запрос без идентификаторов
- **WHEN** пользователь отправляет GET запрос на `/api/suppliers/compare` без параметра `ids`
- **THEN** система возвращает статус 400 с сообщением об ошибке

### Requirement: Ограничение количества сравниваемых поставщиков
Система MUST ограничивать максимальное количество поставщиков для сравнения до 5.

#### Scenario: Превышение лимита
- **WHEN** пользователь отправляет запрос на сравнение 6 поставщиков `/api/suppliers/compare?ids=1,2,3,4,5,6`
- **THEN** система возвращает статус 400 с сообщением о превышении лимита

### Requirement: Визуальное сравнение на фронтенде
Фронтенд SHALL отображать выбранных поставщиков в виде таблицы сравнения, где строки — параметры (название, город, цена, мин. заказ, доставка, сертификаты), а столбцы — поставщики.

#### Scenario: Отображение таблицы сравнения
- **WHEN** пользователь переходит на страницу `/compare?ids=1,2,3`
- **THEN** система загружает данные поставщиков и отображает таблицу с параллельным сравнением всех ключевых параметров

#### Scenario: Пустое сравнение
- **WHEN** пользователь переходит на страницу `/compare` без выбранных поставщиков
- **THEN** отображается сообщение "Выберите поставщиков для сравнения" и ссылка на каталог

### Requirement: Добавление поставщика к сравнению из каталога
Фронтенд SHALL позволять добавлять поставщиков в список сравнения прямо из каталога.

#### Scenario: Добавление через кнопку
- **WHEN** пользователь нажимает кнопку "Сравнить" на карточке поставщика в каталоге
- **THEN** поставщик добавляется в список сравнения (отображается индикатор количества выбранных), максимум 5

#### Scenario: Попытка добавить 6-го поставщика
- **WHEN** пользователь нажимает "Сравнить" и уже выбрано 5 поставщиков
- **THEN** отображается уведомление "Максимум 5 поставщиков для сравнения"
