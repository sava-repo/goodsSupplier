## ADDED Requirements

### Requirement: Полнотекстовый поиск по названию
Система SHALL позволять искать поставщиков по названию с частичным совпадением (case-insensitive).

#### Scenario: Поиск с частичным совпадением
- **WHEN** пользователь отправляет GET запрос на `/api/suppliers/search?search=свежие`
- **THEN** система возвращает всех поставщиков, в названии которых содержится "свежие" (без учёта регистра)

#### Scenario: Поиск без результатов
- **WHEN** пользователь отправляет GET запрос на `/api/suppliers/search?search=несуществующийпоставщик`
- **THEN** система возвращает пустой массив поставщиков и статус 200

### Requirement: Фильтрация по категории
Система SHALL позволять фильтровать поставщиков по одной или нескольким категориям.

#### Scenario: Фильтрация по одной категории
- **WHEN** пользователь отправляет GET запрос на `/api/suppliers/search?category_id=1`
- **THEN** система возвращает только поставщиков, относящихся к категории с id=1

#### Scenario: Фильтрация по нескольким категориям
- **WHEN** пользователь отправляет GET запрос на `/api/suppliers/search?category_id=1,3`
- **THEN** система возвращает поставщиков, относящихся к категории 1 ИЛИ категории 3

### Requirement: Фильтрация по городу и региону
Система SHALL позволять фильтровать поставщиков по городу и/или региону с частичным совпадением.

#### Scenario: Фильтрация по городу
- **WHEN** пользователь отправляет GET запрос на `/api/suppliers/search?city=Москва`
- **THEN** система возвращает только поставщиков, у которых город содержит "Москва"

#### Scenario: Фильтрация по региону
- **WHEN** пользователь отправляет GET запрос на `/api/suppliers/search?region=Московская`
- **THEN** система возвращает поставщиков, у которых регион содержит "Московская"

### Requirement: Комбинированная фильтрация
Система SHALL позволять комбинировать несколько фильтров одновременно (поиск по названию + категория + город).

#### Scenario: Поиск с несколькими фильтрами
- **WHEN** пользователь отправляет GET запрос на `/api/suppliers/search?search=мясо&category_id=1&city=Москва`
- **THEN** система возвращает поставщиков, у которых название содержит "мясо" И категория id=1 И город содержит "Москва"

#### Scenario: Применение фильтров без поискового запроса
- **WHEN** пользователь отправляет GET запрос на `/api/suppliers/search?category_id=2&city=Казань` без параметра `search`
- **THEN** система возвращает всех поставщиков из Казани категории 2

### Requirement: Пагинация результатов поиска
Система MUST поддерживать пагинацию для результатов поиска с параметрами `page` и `per_page`.

#### Scenario: Первая страница результатов
- **WHEN** поиск возвращает 25 результатов и пользователь запрашивает `/api/suppliers/search?search=продукт&page=1&per_page=10`
- **THEN** система возвращает первые 10 результатов и метаинформацию: total=25, page=1, per_page=10, total_pages=3
